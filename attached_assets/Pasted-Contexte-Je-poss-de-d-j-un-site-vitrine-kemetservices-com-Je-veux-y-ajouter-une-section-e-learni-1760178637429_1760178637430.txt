Contexte :
Je possède déjà un site vitrine kemetservices.com. Je veux y ajouter une section e-learning asynchrone pour des formations destinées aux pharmaciens et aux auxiliaires (Côte d’Ivoire). L’objectif : que les apprenants puissent s’inscrire, payer, suivre des leçons vidéo, passer des quiz, télécharger une attestation PDF, et que moi (admin) je puisse créer/éditer les cours et suivre les stats.

Langue par défaut : français (i18n prêt pour ajouter l’anglais plus tard).

Fuseau horaire : Africa/Abidjan.

Design simple, propre, mobile-first, couleurs KEMET (or #DAA520, noir, blanc).

Je veux un module indépendant (monorepo) que je pourrai déployer tel quel puis relier à mon domaine existant (sous-chemin /formations ou sous-domaine).

Base de données SQLite en dev, PostgreSQL en prod.

Pas de services externes payants obligatoires (sauf email/PDF/stockage gratuit ou peu coûteux).

Stockage vidéo : on intègre par URL (Vimeo/YouTube privé, ou fichier .mp4 hébergé), pas d’upload lourd dans la Vercel free tier.

Branding : logo KEMET toujours visible.

7) Schéma Prisma (indicatif)
model User {
  id        String   @id @default(cuid())
  name      String?
  email     String?  @unique
  image     String?
  role      Role     @default(STUDENT)
  enrollments Enrollment[]
  progresses Progress[]
  results   QuizResult[]
  createdAt DateTime @default(now())
}

enum Role { ADMIN TRAINER STUDENT }

model Course {
  id          String   @id @default(cuid())
  slug        String   @unique
  title       String
  audience    String   // "Pharmaciens" | "Auxiliaires"
  synopsis    String
  priceCFA    Int
  coverUrl    String?
  published   Boolean  @default(false)
  modules     Module[]
  resources   Resource[]
  createdAt   DateTime @default(now())
}

model Module {
  id        String  @id @default(cuid())
  title     String
  order     Int
  courseId  String
  course    Course  @relation(fields: [courseId], references: [id])
  lessons   Lesson[]
}

model Lesson {
  id        String  @id @default(cuid())
  title     String
  order     Int
  videoUrl  String? // Vimeo/YouTube/private URL
  contentMd String  // résumé/texte
  moduleId  String
  module    Module  @relation(fields: [moduleId], references: [id])
  quiz      Quiz?
}

model Quiz {
  id        String   @id @default(cuid())
  lessonId  String   @unique
  lesson    Lesson   @relation(fields: [lessonId], references: [id])
  questions Question[]
  passing   Int      @default(70) // % requis
}

model Question {
  id       String  @id @default(cuid())
  quizId   String
  quiz     Quiz    @relation(fields: [quizId], references: [id])
  prompt   String
  options  String  // JSON string array
  answerIx Int     // index de la bonne réponse
}

model Enrollment {
  id        String   @id @default(cuid())
  userId    String
  courseId  String
  paid      Boolean  @default(false)
  paidAt    DateTime?
  user      User     @relation(fields: [userId], references: [id])
  course    Course   @relation(fields: [courseId], references: [id])
}

model Progress {
  id        String   @id @default(cuid())
  userId    String
  lessonId  String
  completed Boolean  @default(false)
  completedAt DateTime?
  user      User     @relation(fields: [userId], references: [id])
  lesson    Lesson   @relation(fields: [lessonId], references: [id])
}

model QuizResult {
  id        String   @id @default(cuid())
  userId    String
  quizId    String
  scorePct  Int
  takenAt   DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id])
}

model Resource {
  id        String  @id @default(cuid())
  courseId  String
  title     String
  url       String
  course    Course  @relation(fields: [courseId], references: [id])
}

model Order {
  id         String   @id @default(cuid())
  userId     String
  courseId   String
  amountCFA  Int
  status     String   // pending | paid | failed
  provider   String   // stripe | orangemoney | mtnmomo
  reference  String?
  createdAt  DateTime @default(now())
}

💳 Paiement Wave — Exigences fonctionnelles
1) Flow utilisateur

Sur la page d’un cours → bouton “S’inscrire / Payer avec Wave”

Création d’une Order (status=pending) en DB.

Redirection vers Wave Checkout (ou affichage d’un Wave Pay Link intégré).

À la réussite du paiement côté Wave → webhook appelle notre API → on valide et on passe l’Order à paid, on crée/active l’Enrollment.

L’apprenant voit aussitôt le cours dans /learning.

2) Endpoints à implémenter

POST /api/payments/wave/checkout

Entrée : { courseId }

Action : crée une Order pending, appelle l’API Wave pour générer un checkout link (ou un paiement collect), renvoie { paymentUrl } pour redirection.

Stocke dans l’Order : wavePaymentId, reference, amountCFA.

POST /api/payments/wave/webhook

Sécurisé par signature HMAC (voir env ci-dessous).

Payload attendu (exemple générique) :

{
  "event": "payment.succeeded",
  "data": {
    "payment_id": "wave_abc123",
    "reference": "ORD_20251011_0001",
    "amount": 35000,
    "currency": "XOF",
    "status": "succeeded",
    "msisdn": "+225XXXXXXXX",
    "timestamp": 1697030400
  },
  "signature": "hex_hmac_sha256_signature"
}


Vérifier la signature → idempotence (ne pas retraiter deux fois).

Si succeeded : Order.status=paid, paidAt=now(), créer/activer Enrollment, envoyer email de confirmation.

GET /api/payments/wave/order-status?reference=...

Optionnel pour polling client side (fallback si le webhook tarde).

Si l’API Wave en CI nécessite un agrégateur (ex : CinetPay/PayTech/Flutterwave avec rail Wave), implémenter l’adapter Wave de façon agnostique (interface WaveGateway) et documenter dans le README où brancher l’agrégateur (URL, clés, signature). L’UX et la DB restent identiques.
